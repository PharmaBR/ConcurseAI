2. Ambiente de Teste

🌐 URL de produção: https://concurseai.dev (ou o IP fornecido pelo professor)

📧 Conta de teste: crie uma conta real com seu e-mail — cada testador deve usar credenciais próprias

📄 Edital de teste: utilize o texto de amostra disponibilizado pelo professor, ou copie um edital público de gov.br

🔑 Pré-requisito: ter acesso à internet e um navegador moderno (Chrome 110+, Firefox 115+, Safari 16+)

⚠️ Atenção: As chamadas de IA têm custo. Execute cada caso de teste uma única vez, de forma deliberada. Evite múltiplas gerações do mesmo conteúdo.

Autenticação
3 casos de teste
TC-AUTH-01 Cadastro de novo usuário Alta
Pré-condições

Usuário não possui conta cadastrada. Navegador sem sessão ativa.

Passos
  1. Acesse a URL da aplicação
  2. Clique em Cadastrar ou navegue para /register
  3. Preencha nome, e-mail válido e senha (mín. 8 caracteres)
  4. Clique em Criar conta
Resultado Esperado
Usuário é redirecionado para a tela principal; a sessão é iniciada automaticamente sem necessidade de login adicional.
Severidade (se falhou)
TC-AUTH-02 Login com credenciais válidas Alta
Pré-condições

Conta cadastrada conforme TC-AUTH-01. Sessão encerrada (logout realizado).

Passos
  1. Acesse /login
  2. Insira e-mail e senha cadastrados
  3. Clique em Entrar
Resultado Esperado
Usuário é autenticado e redirecionado à tela principal. Nenhuma mensagem de erro é exibida.
Severidade (se falhou)
TC-AUTH-03 Acesso bloqueado sem autenticação Média
Pré-condições

Nenhuma sessão ativa (logout ou aba anônima).

Passos
  1. Tente acessar diretamente uma URL de trilha, ex.: /trilha/qualquer-id
  2. Tente clicar em "Salvar" ou "Gerar trilha" na listagem sem estar logado
Resultado Esperado
Usuário é redirecionado para /login ou recebe mensagem solicitando autenticação. Nenhum conteúdo protegido é exibido.
Severidade (se falhou)

Gestão de Concursos
4 casos de teste
TC-CONC-01 Listagem e filtro de concursos Média
Pré-condições

Usuário autenticado. Pelo menos um concurso cadastrado pelo administrador.

Passos
  1. Acesse a tela principal
  2. Verifique se os cards de concursos são exibidos com órgão, cargo, banca e status
  3. Utilize o filtro de status (ex.: "Aberto") e clique em Filtrar
  4. Utilize o filtro de área (ex.: "Federal") e verifique os resultados
Resultado Esperado
Cards exibem informações corretas; filtros reduzem a lista de acordo com os critérios selecionados. Concursos sem edital exibem aviso âmbar.
Severidade (se falhou)
TC-CONC-02 Salvar e remover concurso dos favoritos Média
Pré-condições

Usuário autenticado. Pelo menos um concurso listado.

Passos
  1. Clique no ícone ♡ em um card de concurso
  2. Verifique se o ícone muda para ♥ (vermelho)
  3. Recarregue a página — o concurso deve permanecer salvo
  4. Clique novamente em ♥ para remover
  5. Verifique se o ícone volta a ♡
Resultado Esperado
Estado de favorito persiste entre recarregamentos. Adicionar e remover funcionam sem erros.
Severidade (se falhou)
TC-CONC-03 Criar concurso pessoal com edital colado Alta
Pré-condições

Usuário autenticado. Ter em mãos o texto de um edital (PDF copiado ou texto de amostra).

Passos
  1. Na tela principal, clique em + Adicionar meu concurso
  2. Preencha: Órgão, Cargo, Área
  3. Preencha o campo Banca com "CESPE" (opcional)
  4. Cole o texto do edital no textarea (mínimo 100 caracteres)
  5. Observe o contador de caracteres ficar verde
  6. Clique em ✨ Criar e gerar trilha
  7. Aguarde o redirecionamento para a trilha
Resultado Esperado
Modal fecha; tela mostra geração em progresso; após 15–60s o usuário é redirecionado para a trilha com módulos gerados pela IA. O concurso aparece com badge "meu" na listagem.
Severidade (se falhou)
TC-CONC-04 Validação: edital muito curto deve ser rejeitado Média
Pré-condições

Usuário autenticado. Modal de criação de concurso aberto.

Passos
  1. Preencha Órgão e Cargo
  2. No campo de edital, digite apenas "teste" (menos de 100 caracteres)
  3. Observe se o botão Criar permanece desabilitado
  4. Tente submeter o formulário (se possível)
Resultado Esperado
Botão desabilitado enquanto o texto tiver menos de 100 caracteres. Se submetido via API diretamente, retorna erro 400 com mensagem explicativa.
Severidade (se falhou)

Trilha de Estudos
3 casos de teste
TC-TRL-01 Geração de trilha a partir de concurso existente Alta
Pré-condições

Usuário autenticado. Concurso com edital cadastrado (admin ou próprio). Nenhuma trilha gerada para este concurso.

Passos
  1. Localize o card do concurso desejado
  2. Clique em ✨ Gerar trilha de estudos
  3. Observe o spinner e contador de segundos durante a geração
  4. Aguarde o redirecionamento automático
Resultado Esperado
Após geração (10–90s), a trilha exibe: header com órgão + cargo + banca; lista de módulos com nome e tópicos; banner de boas-vindas orientando o próximo passo.
Severidade (se falhou)
TC-TRL-02 Marcar subtópico como estudado e verificar progresso Média
Pré-condições

Trilha gerada com pelo menos um módulo expandido.

Passos
  1. Expanda um módulo clicando nele
  2. Expanda um tópico para ver os subtópicos
  3. Marque dois ou três subtópicos com o checkbox ✓
  4. Observe a barra de progresso do tópico atualizar
  5. Recarregue a página e verifique se as marcações persistem
Resultado Esperado
Checkboxes permanecem marcados após recarga. Barra de progresso reflete a porcentagem correta de subtópicos concluídos.
Severidade (se falhou)
TC-TRL-03 Retorno ao concurso mostra "Continuar estudando" Baixa
Pré-condições

Trilha já gerada para um concurso.

Passos
  1. Volte à tela principal
  2. Localize o card do concurso que já possui trilha
Resultado Esperado
O botão do card exibe "▶ Continuar estudando" (verde) em vez de "✨ Gerar trilha". Clicar redireciona para a trilha existente.
Severidade (se falhou)

Chat IA com Memória
3 casos de teste
TC-CHAT-01 Primeira pergunta sobre tópico do módulo Alta
Pré-condições

Trilha gerada. Módulo expandido. Chat visível no card do módulo.

Passos
  1. Clique no botão 💬 Chat IA no card do módulo
  2. Digite uma pergunta relacionada ao conteúdo do módulo (ex.: "Explique o conceito de X")
  3. Pressione Enter ou clique em Enviar
  4. Observe o streaming de tokens chegando
Resultado Esperado
Balão do usuário aparece à direita imediatamente; indicador de digitação (3 pontos) aparece; resposta chega em streaming com formatação markdown. Cursor pulsante visível durante geração.
Severidade (se falhou)
TC-CHAT-02 Memória conversacional — pergunta de acompanhamento Alta
Pré-condições

TC-CHAT-01 executado com sucesso. Thread de chat visível com pelo menos uma troca.

Passos
  1. Sem limpar o chat, envie uma segunda pergunta que referencia a resposta anterior (ex.: "Você pode aprofundar o segundo ponto que mencionou?")
  2. Observe a resposta
  3. Envie uma terceira pergunta de acompanhamento
Resultado Esperado
A IA demonstra memória da conversa: menciona ou aprofunda o conteúdo das respostas anteriores sem pedir que o usuário repita o contexto. O histórico completo é visível na thread.
Severidade (se falhou)
TC-CHAT-03 Restrição de escopo — pergunta fora do módulo Alta
Pré-condições

Chat aberto em qualquer módulo.

Passos
  1. Envie uma pergunta completamente fora do escopo do módulo (ex.: "Qual a capital da França?", "Me dê uma receita de bolo", "Quem é o presidente do Brasil?")
  2. Observe a resposta da IA
Resultado Esperado
A IA recusa responder o assunto fora do escopo, explica que só pode tratar dos tópicos do módulo, e oferece ajuda com o conteúdo correto. Não fornece a resposta solicitada.
Severidade (se falhou)

Quiz Adaptativo
4 casos de teste
TC-QUIZ-01 Quiz direto por subtópico (botão inline) Alta
Pré-condições

Módulo expandido com pelo menos um tópico e subtópicos visíveis.

Passos
  1. Expanda um tópico para ver seus subtópicos
  2. Passe o mouse sobre um subtópico — o ícone 📝 deve aparecer
  3. Clique no ícone 📝 ao lado do subtópico
  4. O modal de quiz deve abrir diretamente nas perguntas (sem seletor de nível)
  5. Responda as 5 questões
  6. Clique em Ver resultado
Resultado Esperado
Quiz inicia imediatamente com 5 questões sobre aquele subtópico específico. Resultado exibe pontuação e indica questões corretas/erradas.
Severidade (se falhou)
TC-QUIZ-02 Quiz geral do módulo via botão principal Alta
Pré-condições

Módulo com tópicos carregados.

Passos
  1. Clique em 📝 Quiz no card do módulo (botão principal)
  2. No modal, selecione o nível Módulo
  3. Clique em Gerar quiz
  4. Responda todas as questões e submeta
Resultado Esperado
5 questões abrangendo diferentes tópicos do módulo. Após submissão, mostra pontuação e ativa o botão de flashcards se houver erros.
Severidade (se falhou)
TC-QUIZ-03 Toggle "Só o que estudei" filtra questões Alta
Pré-condições

Pelo menos 2 subtópicos marcados como estudados (checkbox ✓). Modal de quiz aberto no nível Tópico ou Módulo.

Passos
  1. Abra o modal de quiz pelo botão do módulo
  2. Selecione nível Tópico ou Módulo
  3. Ative o toggle Só o que estudei (verde)
  4. Clique em Gerar quiz
  5. Observe se as questões se referem apenas aos subtópicos marcados
Resultado Esperado
As questões geradas estão restritas ao conteúdo dos subtópicos que o usuário marcou como estudados. Não aparecem perguntas sobre subtópicos não estudados.
Severidade (se falhou)
TC-QUIZ-04 Resultado do quiz exibe gabarito e estrelas Média
Pré-condições

Quiz completo com pelo menos uma resposta errada.

Passos
  1. Complete um quiz errando ao menos uma questão intencionalmente
  2. Clique em Ver resultado
  3. Verifique a tela de resultado
  4. Observe o card do módulo após fechar o modal
Resultado Esperado
Resultado exibe pontuação (ex.: 3/5), estrelas de desempenho, gabarito de cada questão com explicação, e botão "Gerar flashcards para o que errei". Card do módulo exibe indicador de estrelas.
Severidade (se falhou)

Flashcards por Lacuna
2 casos de teste
TC-FLASH-01 Gerar flashcards após quiz com erros Alta
Pré-condições

TC-QUIZ-04 executado com pelo menos uma resposta errada. Tela de resultado do quiz aberta.

Passos
  1. Na tela de resultado, clique em Gerar flashcards para o que errei
  2. Aguarde a geração (5–30s)
  3. Observe os flashcards gerados
Resultado Esperado
Flashcards apresentam frente (conceito/termo) e verso (explicação/definição) relacionados especificamente às questões erradas. Cada card aborda a lacuna identificada.
Severidade (se falhou)
TC-FLASH-02 Navegação e revisão de flashcards Média
Pré-condições

Flashcards gerados (TC-FLASH-01). Também pode ser acessado via botão 📚 Flashcards no card do módulo.

Passos
  1. Abra o modal de flashcards
  2. Leia a frente do primeiro card
  3. Clique ou vire o card para ver o verso
  4. Navegue para o próximo card com o botão →
  5. Navegue de volta com o botão ←
  6. Verifique o contador (ex.: "2 / 5")
Resultado Esperado
Navegação entre cards funciona sem erros. Frente e verso exibem conteúdo distinto e complementar. Contador atualiza corretamente.
